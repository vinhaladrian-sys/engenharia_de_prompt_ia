Relatório de Projeto: Módulo 3 – Low-Code / No-Code / Vibecode
🐱 Desafio Escolhido
O desafio consistiu na criação do site para um restaurante temático com foco em gatos (Cat Café). O objetivo do espaço é permitir que os clientes desfrutem de comidas e bebidas personalizadas enquanto aproveitam a companhia de felinos, proporcionando uma experiência acolhedora, interativa e diferenciada.

🛠️ O Protótipo: Gattini - Cat Café Fofo
Prompt Base Utilizado
"Estamos desenvolvendo o Projeto Módulo 3 da disciplina 'Engenharia de Prompt e Aplicações em AI' da UDF, baseado no conceito de Vibecode. Vamos criar o site Gattini - Cat Café Fofo, um site interativo e temático com visual fofo, contendo: home atrativa, cardápio completo, galeria de gatos, sistema de reservas, feedback com estrelas, animações (gatos caminhando, confetes, loading screen), modal de sucesso e responsividade."

Repositório do Protótipo: Os arquivos estruturados do site foram organizados e enviados para uma pasta dedicada no GitHub.

📊 Análise do Processo de Desenvolvimento
Vantagens Identificadas
Automação de Arquivos: A IA (especialmente a Manus AI) otimizou o fluxo de trabalho ao entregar o projeto estruturado diretamente em uma pasta compactada (.zip).

Iteração Fluida e Ágil: Pequenas mudanças e ajustes na descrição do prompt geravam novas versões do projeto rapidamente, eliminando a necessidade de reescrever o código do zero.

Velocidade de Entrega: O tempo de resposta das IAs na geração das estruturas de código com base nos prompts criados foi extremamente alto.

Limitações Encontradas
Instabilidade no Output: Apesar de entregarem os arquivos na estrutura correta, as ferramentas frequentemente apresentavam erros de execução no resultado final do código.

Dependência de Assets Externos: As plataformas não foram capazes de criar animações complexas e efeitos sonoros nativamente do zero, exigindo a busca e vinculação de arquivos externos.

Desalinhamento de Escopo: Mesmo fornecendo prompts altamente detalhados, as IAs por vezes falhavam em captar a real necessidade e a nuance estética esperada pelo grupo.

🧠 Reflexão Crítica do Grupo
O grupo enfrentou um desafio inicial complexo na definição e afunilamento do tema para o site. Após debates, a escolha pelo Cat Café se mostrou ideal por permitir explorar a interatividade.

Durante o desenvolvimento técnico, surgiram barreiras significativas na inserção de caminhos de imagens, áudios no código e na vinculação de efeitos sonoros aos cliques dos botões. Houve inclusive debates internos sobre a viabilidade de manter ou descartar esses recursos sonoros.

A Solução: Para mitigar as falhas da IA, o grupo assumiu o controle analítico: realizamos o download manual dos assets de imagem e som, corrigimos a estrutura de diretórios e integramos os caminhos diretamente no código de forma adequada, garantindo a fidelidade da experiência do usuário (UX).

🚀 Melhorias Sugeridas para o Protótipo (Pós-Feedback)
Com base na discussão em sala de aula, o grupo listou as seguintes evoluções para as próximas versões do Gattini:

Refatoração de Áudio: Implementar um controle de volume ou botão de mutar visível para que os sons de clique não se tornem repetitivos ou incômodos ao usuário.

Otimização de Carregamento (Performance): Comprimir as imagens dos gatinhos e os arquivos de áudio para garantir que a loading screen criada pela IA seja rápida e funcional.

Fallback de Mídia: Tratar o código para que, caso uma imagem externa falhe em carregar, o site exiba um ícone fofo padrão (placeholder) em vez de quebrar o layout.
