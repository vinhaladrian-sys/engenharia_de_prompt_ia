Projeto Módulo 3 – Low Code/No Code/Vibecode 
Desafio Escolhido 
O desafio foi criar um site para um restaurante temático com foco em gatos, onde os clientes pudessem desfrutar de comidas e bebidas enquanto aproveitam a companhia dos gatos, proporcionando uma experiência acolhedora e diferenciada.  
Protótipo: 
Designado uma pasta no Github com os arquivos do site para o protótipo: 
Prompt: Estamos desenvolvendo o Projeto Módulo 3 da disciplina "Engenharia de Prompt e Aplicações em AI" da UDF, baseado no conceito de Vibecode. Vamos criar o site Gattini - Cat Café Fofo, um site interativo e temático com visual fofo, contendo: home atrativa, cardápio completo, galeria de gatos, sistema de reservas, feedback com estrelas, animações (gatos caminhando, confetes, loading screen), modal de sucesso e responsividade.

 Plataforma Utilizada: 
Nome das plataformas: Gemini, chatgpt , Grok, Manus e copilot  
Justificativa das IA´s : cada uma teve sua finalidade para criar o site e ferramentas para cada situação  
Vantagens Identificadas 
1	– A IA (Manus AI) já envia os arquivos em uma pasta zip  
2	-  Fácil iteração: pequenas mudanças na descrição do prompt geram novas versões do projeto rapidamente, sem necessidade de escrever do zero. 
3-  Tem uma velocidade extrema que as AI entregam de acordo com prompt criado. Limitações Encontradas 
1	-  Ocorre muitos erros  no resultado mesmo entregando os arquivos adequado 
2	– O app não é capaz de criar animações e sons do zero, sendo necessário anexar arquivos externos 
3	– Apesar de pedimos detalhadamente o site para AI, não conseguem entrar realmente o que precisamos. Reflexão Crítica 
O grupo teve grande dificuldade em escolher um tema específico para o desenvolvimento do site. Após várias discussões, decidimos criar um restaurante temático com foco em gatinhos, oferecendo comidas e bebidas simples. 
Durante a criação do site, enfrentamos alguns problemas técnicos, principalmente relacionados à inserção de imagens e áudios no código, além da configuração dos sons para cada clique dos botões. Também houve dúvidas sobre manter ou não esses recursos no projeto final. 
Para solucionar esses problemas, realizamos o download das imagens e dos arquivos de áudio, fazendo o upload correto para o site e integrando-os ao código de forma adequada. Assim, conseguimos melhorar o funcionamento e a experiência do usuário no projeto.
